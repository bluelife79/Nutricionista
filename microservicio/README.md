# Microservicio de Re-ranking Semántico

## Setup

```bash
# Install dependencies
cd microservicio
uv sync

# Generate embeddings (run once, or after database.json changes)
uv run python ../scripts/embed_foods.py

# Start the service
uv run uvicorn main:app --reload --port 8000
```

## Endpoints

- `GET /health` — status, n_foods loaded
- `POST /rerank` — semantic re-ranking of food candidates

### POST /rerank — request / response

```json
POST /rerank
{
  "query": "espinaca",
  "candidates": [
    { "id": "food_123", "tier": 1 },
    { "id": "food_456", "tier": 2 }
  ]
}
```

```json
{
  "ranked": [
    { "id": "food_123", "tier": 1, "score": 0.87 },
    { "id": "food_456", "tier": 2, "score": 0.65 }
  ],
  "missing": [],
  "model": "paraphrase-multilingual-MiniLM-L12-v2"
}
```

Candidates absent from the embedding index are returned in `missing` and excluded from `ranked` (HTTP 200 is still returned).

## Environment variables

- `ALLOWED_ORIGIN` — additional CORS origin (e.g. `https://tu-app.vercel.app`)

## Deploy (Railway / Render)

Set start command: `uvicorn main:app --host 0.0.0.0 --port $PORT`

## Notes

- Embeddings are pre-generated by `scripts/embed_foods.py` and stored in `data/`.
- At startup, the service checks that `database.json` hash matches `data/meta.json`. If there is a mismatch, a warning is logged and you should re-run the embed script.
- The first request after startup is warm — model loading happens inside the lifespan handler before the server accepts connections.
